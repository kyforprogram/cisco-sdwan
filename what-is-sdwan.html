Cisco SD-WANとは？ ViptelaからSASEへ

Viptelaの買収とCisco SD-WANの誕生

Cisco SD-WANの物語は、2017年にCiscoがSD-WANのパイオニアであったViptela社を買収したことから始まります。Viptelaは、その先進的なアーキテクチャと強力なルーティング機能で高く評価されていました。Ciscoはこの優れた技術を取り込み、自社の広範なネットワーク製品群と統合することで、エンタープライズ向けの強力なSD-WANソリューションを確立しました。これが現在の「Cisco SD-WAN（旧Viptelaソリューション）」です。

このため、Cisco SD-WANのアーキテクチャやコンポーネントの名称（vManage, vSmartなど）には、Viptelaの思想が色濃く反映されています。

単なるSD-WANを超えて：SASEへの道

Cisco SD-WANは、単に拠点間を接続するだけのソリューションではありません。Ciscoは、ネットワーク機能とセキュリティ機能をクラウド上で統合して提供する新しいアーキテクチャ、「SASE（サシー、Secure Access Service Edge）」の実現を視野に入れています。

SASEを構成する主要な要素

  • SWG (Secure Web Gateway): 不正なWebサイトへのアクセスをブロックし、Webトラフィックを保護します。
  • CASB (Cloud Access Security Broker): クラウドアプリケーションの利用状況を可視化・制御し、データセキュリティを強化します。
  • FWaaS (Firewall as a Service): クラウドベースで提供されるファイアウォール機能で、どこからでも一貫したセキュリティポリシーを適用します。
  • ZTNA (Zero Trust Network Access): ユーザーやデバイスがネットワークにアクセスする際に、常に認証・認可を行うことで、信頼しないアクセスを排除します。

SASEの世界では、SD-WANは安全で最適なネットワーク経路を提供する「基盤」の役割を担います。そして、その上でファイアウォール、セキュアウェブゲートウェイ（SWG）、CASB（Cloud Access Security Broker）といった様々なセキュリティ機能がクラウドサービスとして提供されます。これにより、ユーザーがどこから（オフィス、自宅、外出先）、どのクラウドサービスにアクセスしようとも、一貫した高いレベルのセキュリティを確保できるようになります。

Cisco SD-WANを導入することは、将来的にSASEアーキテクチャへとスムーズに移行するための重要な第一歩となるのです。

Cisco SD-WANアーキテクチャの概要

Cisco SD-WANは、ネットワークの役割を以下の4つの「プレーン（層）」に明確に分離・定義しているのが大きな特徴です。

  • オーケストレーションプレーン（vBond）SD-WAN Validator: 新しい機器の参加を調整し、全体をつなぎ合わせる役割。
  • マネジメントプレーン（vManage）SD-WAN Manager: ネットワーク全体を監視・管理する役割。
  • コントロールプレーン（vSmart）SD-WAN Controller: 通信経路を決定し、ポリシーを制御する「頭脳」の役割。
  • データプレーン（SD-WAN vEdge cEdge）: 実際にデータを転送する役割。

このアーキテクチャにより、従来のようにルーター一台一台にログインして設定を行う必要がなくなり、中央の管理ポイントからネットワーク全体を柔軟かつ効率的に制御することが可能になります。
